# Peacock 移动端 UI 规范

> Android 端请严格参照此规范实现，保持与 iOS 版本视觉一致。

## 设计原则

- **原生感**：遵循各平台的设计语言（Material 3 / iOS HIG），但保持视觉统一
- **简洁**：不要花哨装饰，干净的留白和清晰的层级
- **一致**：iOS 和 Android 的颜色、间距、字号保持一致

```
├─────────────────────────────────────┤
│ 保存中... | Spacer | [📋] [↗] [🗑]  │  工具栏

│ 普通文字 [绿色芯片] 继续文字         │  ← 内联渲染
│                                     │
│ 更多内容...                         │
│                                     │
│ [另一个芯片]                        │
│                                     │
├─────────────────────────────────────┤
│ 📝 备注                             │  固定底部, 始终一行
└─────────────────────────────────────┘
```

### 工具栏
- 左侧: 保存状态 (保存中.../已保存), 12sp
- 右侧: 复制(doc.on.doc) + 分享(share) + 删除(trash), 各 16dp 图标

### 内容区 — 快速复制芯片（最关键的组件）

**所见即所得，没有模式切换：**
- 编辑区域始终可编辑
- `[[text]]` 实时渲染为绿色芯片, 方括号不显示
- 芯片与普通文字在同一行内混排
- 普通文字可正常输入/删除
- 芯片不可编辑, 但可整体删除

**芯片样式：**
- 背景: `rgba(13, 148, 136, 0.15)`
- 文字: `rgb(5, 100, 80)`, 13sp medium
- 圆角: 6dp
- 边框: primary 0.3 opacity, 0.5dp
- 水平 padding: 8dp, 垂直 padding: 3dp
- 与前后文字间有 1 个空格的视觉间隔

**交互：**
- **点击芯片** → 复制到剪贴板 → 芯片缩小到 0.85 → 恢复 (0.4秒动画)
- **长按芯片** → 上下文菜单: "复制" + "取消标记"
- **选中文字 → 标记** → 在系统选择菜单加自定义 action "标记为快速复制"
- **删除芯片** → 按退格到芯片边缘时, 整个 `[[text]]` 一起删除
- 取消标记 → 移除 `[[]]`, 芯片变回普通文字

**Android 实现建议：**
- 用 `EditText` + `SpannableStringBuilder`
- 芯片用 `ReplacementSpan` 或 `BackgroundColorSpan` + `ClickableSpan`
- 存储的原始文本始终包含 `[[]]`
- 显示时 parse `[[]]` 并替换为 span

### 备注
- 固定在底部, 不随内容滚动
- 单行, 13sp, 次级文字色
- 左侧小图标 (note icon, 12dp, 三级色)

## 核心提醒

1. **不要过度设计** — 保持简洁，用系统组件
2. **颜色一致** — Primary Teal `#0D9488` 是唯一的强调色，不要引入其他彩色
3. **暗色模式** — 所有自定义颜色都要有暗色版本
4. **触控区域** — 所有可点击元素最小 44x44dp
5. **Plus 面板背景** — 和聊天背景一致，不要用不同颜色
6. **片段预览** — 去掉 `[[]]` 再显示
7. **输入栏对齐** — +号、输入框、发送按钮三者垂直居中
